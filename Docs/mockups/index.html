Agent instance as a chat log

Four ways to render a running agent instance as a conversation — agent messages, tool calls (with input & output), and sub-agents all appear as events. Every event is collapsed by default, showing a one-line summary and live status, and expands on click. Toggle light/dark on any page. Pick a base, then mix the design choices below.

All four use the same live trace: the research / web-report agent (model gemini-2-5-flash) summarizing & fact-checking three articles. It has scraped two pages (one 403 → retried), extracted 4 claims, and is now mid-run inside a fact-checker sub-agent — so you can see done, failed, running, and queued states side by side.

05 · Grouped — steps & cubes

Same-step tools, messages & sub-agents are grouped and collapsed under their step, nested beneath the caller. Built for fan-out: when a group exceeds 5 calls it becomes a grid of status cubes — green done, blue running, red failed, dashed queued.

Scales to 50+ — 50 web-scrapes and 10 sub-agents in one run, no endless scroll.
Radio-select — one cube expands at a time; hover any cube for a status tooltip.
Recursive — a sub-agent cube expands into its own stepped trace, with its own cubes.

Grouped — other looks

Same engine and every feature as the Grouped view above — five visual treatments, from calm & editorial to dense & technical. They all share grouped-core.js, so behaviour is identical; only the styling changes.
